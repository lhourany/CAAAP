# Al Ittihad — CAR Platform (prototype)

A clickable prototype of the Curriculum & Assessment Review (CAR) process.

**Live:** https://lhourany.github.io/CAAAP/

This README explains **how to use** the prototype — where to start, what order to follow, and what each part of the screen is for.

> **It's a demo.** Nothing you type is saved. Refreshing the page resets everything back to the start. The plain black-and-white styling is deliberate placeholder design. Best viewed in a desktop browser.

---

## Where to start

When the page opens you are on **Ongoing Proposals**, with the **Review** tab showing one sample review:
*"Grade 10 Mathematics — Review (sample)."*

**Click that review card to open it.** Everything starts there.

---

## Working a review — follow the tabs in order

The review opens in the **Draft** stage. You can see the stage at the top:

`Draft → Validation Board → AQC → Approved`

The review moves along that bar as you complete each step. Work the tabs **left to right, in order**:

1. **Summary** — the review's basic information (subject, grade, campuses, owner). Read it first.
2. **Course Review** — the heart of the review. Answer each question. For every question you can write a response and/or set a status:
   - ✔ in place  ·  △ needs review  ·  ✘ concern
3. **Action Plan** — for anything you flagged, add an action with **+ Add action**. When you add one, you pin it to the **principle(s)** and **question(s)** it addresses, then give it an owner, a due date, and what "done" looks like.
4. **Attachments** — where supporting evidence is attached.

When **every question has either a written answer or a status**, click **Submit to Validation** (top right). If anything is still missing, it will tell you exactly which questions need attention — you cannot submit an incomplete review.

---

## After you submit

Submitting moves the review to the **Validation Board** stage and opens the **Validation** page:

5. **Validation** — the board reviews your work. Set a judgement (Met / Query / Not met) on each of the five checks, add comments, then either:
   - **Confirm & advance to AQC** (all five checks must have a judgement), or
   - **Return to reviewer** — sends it back with comments, which then appear under the relevant questions in Course Review.
6. **AQC ratification** — the final committee. Set a judgement on each of the three checks, then either:
   - **Ratify & approve** (all three must have a judgement) → the review is **Approved**, or
   - **Return with conditions** — sends it back to the reviewer with the conditions to address.

When approved, the bar at the top shows the full chain complete.

---

## What's on the left, and why

The menu down the left side stays with you the whole time:

- **Dashboard** — the home view (opens your list of proposals).
- **Ongoing Proposals** — your main workspace. This is where reviews live. Across the top it has three tabs: **Review**, **Planning**, and **Full Proposal**.
- **My Tasks** / **Notifications** — placeholders for now; they don't open anything yet.
- **CAAAP Principles** — the set of principles every review is checked against. Open it any time as a reference while you work.
- **Standards Library** — links to the external standards and regulators the curriculum is anchored to.

---

## Why some areas look empty

Some parts are blank **on purpose** — they fill up later in a review's life, not at the start:

- **Planning** and **Full Proposal** tabs are empty until a review is **approved**. Planning is the stage *after* approval (writing the next curriculum); Full Proposal is the evidence record built later. On a fresh review, there is nothing in them yet — that is expected.
- **Attachments** is empty until you add evidence.
- **My Tasks** and **Notifications** are placeholders for a future version.

So if a tab is empty, it usually means *"this comes later in the process,"* not that something is broken.

---

## The short version

1. Open the sample review.
2. Work the tabs in order: **Summary → Course Review → Action Plan → Attachments.**
3. **Submit to Validation** (only works once every question is answered).
4. On the **Validation** page, judge the five checks → **advance to AQC**.
5. On the **AQC** page, judge the three checks → **Ratify & approve**.
6. Done — the review is **Approved**.
